- **初始兵力**：等於大地遊戲期間四次世界史更新（U1–U4）**累積的兵力總和**。詳細兵力獲得規則見[總規則說明書](https://hackmd.io/d8TBxXMrR7ysTYJ21VPpcA)第六節。

    - 持有多領地國家將兵力平均分散，採無條件進位。
    - **未持有領地**的隊伍從中立區域出發。

## 三、回合操作

每個回合（7 分鐘）最多可執行 **3 次操作**，所有操作**平行執行**（同一回合內的操作互不阻擋）

---

- 特殊判定：未被佔領的國家視為他國領地，以避免多國家同時使用 `moving` 操作。
- 若有國家在場面上總兵力為 0，強質移動到中立區域養兵。

---

### 操作一：移動兵力

```
moving(S, E, n)
```

| 參數 | 說明 |
| :---- | :---- |
| S | 出發區域（必須為己方領地、中立區域或資源點）|
| E | 目的區域（同上限制）|
| n | 移動兵力數（正整數，不超過 S 當前兵力）|

- **不可**移動至他國領地。

---

### 操作二：進攻

```
attack(S, E, n)
```

| 參數 | 說明 |
| :---- | :---- |
| S | 出發區域（必須為己方領地、中立區域或資源點）|
| E | 目標（必須為**他國領地**）|
| n | 派遣兵力數（正整數，不超過 S 當前兵力） |

- 若 E 為中立區域或資源點，視為違反世界法，S 損失 n 兵力（自殺）。
- 可以發動多個 attack 從不同區域同時進攻同一個 E。

---

### 操作三：聯盟（防禦型）

```
union(S, P, E, n)
```

| 參數 | 說明 |
| :---- | :---- |
| S | 出發區域 |
| P | 聯盟夥伴國（單一隊伍代號）|
| E | 目標領地（E 為 P 國領地時為請求方；E 為我國領地時為接受方）|
| n | 派遣／接受的兵力數 |

| 情況 | 說明 |
| :---- | :---- |
| E 為 P 國領地 | 請求加入聯盟，在 P 同意時，將 S 的 n 兵力駐守於 E |
| E 為我國領地 | 接受 P 國的聯盟邀請，同意 P 轉移 S 的 n 兵力駐守於 E |

**聯盟成立條件（缺一不可）：**
1. 雙方都發出 union
2. 雙方設定的 S、E、n **嚴格相同**，P 互指對方

example:
> 有國家 A, B，分別擁有 D_A, D_B 領地，並且 D_A 擁有至少 n 兵力。
> `A: union(D_A, B, D_B, n)`
> `B: union(D_A, A, D_B, n)`
> => A 成功從 D_A 轉移 n 兵力駐守到 D_B

---

### 操作四：聯合進攻

```
union_attack(S, P, E, n)
```

| 參數 | 說明 |
| :---- | :---- |
| S | 出發區域 |
| P | 聯合進攻國（可為複數，以 `[X,Y]` 格式列出）|
| E | 目標領地（必須為非我國、非 P 國的他國領地）|
| n | 派遣兵力數 |

- 若該聯盟進攻成功，由投入兵力最大者佔領，其餘為駐守。

**合法聯盟條件（最大互信優先原則）：**

聯盟配對以**最大互信優先**為原則，依照以下步驟遞迴決定：

1. **有效聯盟**：一組隊伍中，若每位成員都在自己的 P 列表中指名其他所有成員，則構成有效聯盟。
2. **最大優先**：從所有有效聯盟中，選出成員數最多的那一組。
3. **兵力決勝**：若有多個成員數相同的有效聯盟，取參與兵力（n）總和最高的那組生效；未勝出的隊伍回到剩餘池重新計算。
4. **遞迴拆解**：確定一組聯盟後，從剩餘隊伍重複步驟 1–3，直到無法再形成任何有效聯盟。
5. **單獨進攻**：最終無法配對的隊伍，各自單獨進攻目標。

> **範例 1（三國完整聯盟）：**
> - A: `union_attack(D_A, [B,C], E, 300)`
> - B: `union_attack(D_B, [A,C], E, 200)`
> - C: `union_attack(D_C, [A,B], E, 100)`
> - 分析：有效聯盟 {A,B,C}（三人互相指名），為最大聯盟。
> - 結果：A+B+C 以 600 合力進攻 E。

> **範例 2（不完整互信，兵力決勝）：**
> - A: `union_attack(D_A, [B,C], E, 300)` （A 想跟 B 或 C 合作）
> - B: `union_attack(D_B, [A], E, 200)` （B 只認 A）
> - C: `union_attack(D_C, [A], E, 100)` （C 只認 A）
> - 分析：有效聯盟 {A,B}（A 指名 B，B 指名 A ✓）和 {A,C}（A 指名 C，C 指名 A ✓）。{A,B,C} 無效，因為 B 未指名 C、C 未指名 B。兩組並列最大（大小 2），兵力決勝：A+B=500 > A+C=400，{A,B} 勝出。
> - 結果：A+B 以 500 合力進攻 E；C 單獨以 100 進攻 E。

> **範例 3（兩組平行聯盟 + 遞迴）：**
> - A: `union_attack(D_A, [B,C], E, 300)`
> - B: `union_attack(D_B, [A,C], E, 200)`
> - C: `union_attack(D_C, [A,B], E, 100)`
> - D: `union_attack(D_D, [E,F], E, 150)`
> - E: `union_attack(D_E, [D,F], E, 100)`
> - F: `union_attack(D_F, [D,E], E, 50)`
> - 分析：有效聯盟最大為 {A,B,C}（600）與 {D,E,F}（300），並列大小 3。兵力決勝：A+B+C=600 > D+E+F=300，{A,B,C} 先勝出。移除後剩餘 {D,E,F}，重新計算：{D,E,F} 為唯一最大聯盟，勝出。
> - 結果：A+B+C 以 600 合力進攻 E；D+E+F 以 300 合力進攻 E。

## 四、回合結算規則（照順序）

### 0. 所有操作執行完畢並更新結果狀態。

### 1. 領地重新結算領主

若任一國家對某領地發起進攻（attack 或 union_attack），則：

- **佔領方**：兵力**最大**的單位（單國或聯盟）成為新領主。
- **佔領方獲得兵力**：自身兵力 + 20% × 各失敗方派遣兵力
- **失敗方**：損失全部派遣兵力。

> **範例1：**
> - A（原佔領者）：100 兵力佔領 D_A
> - B：`attack(D_B, D_A, 99)`
> - C：`attack(D_C, D_A, 99)`
> - 結果：B, C 各自以 99 兵力進攻。
> A 維持佔領，最終兵力 = 100 + ⌊99×20%⌋ + ⌊99×20%⌋ = **138**

> **範例2：**
> - A（原佔領者）：100 兵力佔領 D_A
> - B：`union_attack(D_B, [C], D_A, 90)`
> - C：`attack(D_C, [B], D_A, 110)`
> - 結果：B, C 聯合進攻以 90+110 兵力進攻。
> - - 改由 C 佔領，C 兵力 = 110 + ⌊100x20%⌋ = 130
> - - B 在聯合進攻中占較少兵力，為駐守兵力= 90 + ⌊100x20%⌋ = 110
> - - 注意到在 5 國家聯合進攻的情況下，戰爭前後總兵力不變，此情況為壓倒性投降，因此沒有淨兵力損失。非常合理。
> - - 注意到在超過 5 國家聯合進攻的情況下，戰爭後總兵力增加，此情況為世界型聯盟，在征途上會有野生士兵投靠聯盟尋求庇護，因此淨兵力增加。非常合理*2。
> 

### 2. 領地產出（國力）

- 若結算時只有 A 國佔領：A 國國力 +X（X 為該領地每回合產出）
- 若有佔領國 A 與駐守國 B（after `union` or `union_attack`）：
  - A 國獲得：½X + ½X × A的兵力比例
  - B 國（駐守）獲得：½X × B的兵力比例
  - 以此類推

### 3. 中立區域結算

- 駐守 n 兵力的國家：兵力變為 n × 1.5（無條件捨去小數）
- **0 兵力救濟**：兵力為 0 的國家每回合額外獲得 **1000 兵力**（含強制移入中立區域者）

### 4. 資源點結算

- 依各國駐守兵力比例分配資源（含小數無條件捨去）

## 六、操作注意事項
### 1. 操作衝突

若一個回合提交的操作**所需兵力總和超過當前可用兵力**：

1. 涉及違規的**所有**操作視為**無效**（不執行任何移動或攻擊）
3. 涉及違規的領地全數兵力強制移入**中立區域**，並不會領取該回合中立區域獎勵。

> **範例（衝突觸發）**：
> - A 領地當前兵力 100
> - 提交：`moving(A, B, 50)` + `attack(A, C, 80)` → 所需 130 > 100
> - 結果：兩個操作全部無效，A 領地兵力全數移入中立區域。

---

### 2. 單國平行

同一小隊的操作將會偽平行執行。
> 因為軍隊行進需要時間，因此你不能在一回合內先將多個兵力 `moving` 集合後然後再做一次 n 比較大的 `attack`。取而代之的事，你應該由不同領地直接發起多個 `attack`。

若同一回合執行以下操作：
```
moving(A, B, 50)
attack(A, C, 80)
```
由於操作**平行執行**，兩個指令都視為從回合開始時的原始兵力狀態出發。若兵力足夠（例如共 150），則兩個操作均正常執行；若兵力不足（例如只有 100），則觸發衝突裁決，全部操作無效並移入中立區域。

---

### 3. 跨國平行

各小隊的操作將會偽平行執行，並且統一先做移出，再做移入。
> 各國士兵將同時出發，並且恰好同時抵達目的地。

任何兵力轉移，可以視為先移出原本的領地，所有人都移出後，所有人同時抵達目的地，並進行戰爭（如果有觸發的話）。
